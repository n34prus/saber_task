Описание архитектуры:
Для мягкого взаимодействия внедрена шина событий.
Боевая система, компонент здоровья и другие классы отправляют в нее сообщения, а подписчики на них реагируют.
Шина событий построена как Observer с подключенными GameplayTag в качестве каналов для обеспечения унификации.
Боевая система построена поверх системы способностей, которая подключается при помощи компонентов к актору-владельцу.
Урон наносится акторам через стандартный интерфейс нанесения урона класса AActor.
Отправитель урона (способность) формирует пакет, который проходит через модификаторы владельца и цели (через боевую систему), и в имплементацию попадает уже модифицированный урон.
Система статистики не зависит от других модулей кроме шины событий и инкрементирует свои поля при совпадении канала получаемого из шины события.
Частная геймплейная логика (чарактер, аи-контроллер, способности, модификаторы, виджеты и пр) реализована либо в отдельном модуле GameplayImplementation, либо в ассетах проекта.
